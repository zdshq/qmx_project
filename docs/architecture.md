# 架构说明

## 总体流程

整个 Agent 每轮会执行下面几个阶段：

1. 采集系统上下文
2. 抓取屏幕截图
3. 读取最近若干条历史摘要
4. 调用本地多模态模型判断状态
5. 将结果写入 SQLite
6. 清理过期截图
7. 如果到达北京时间 23:00，则生成日报

## 模块职责

### `config.py`

负责读取 `.env` 和环境变量，生成统一的运行配置。

### `capture/`

- `screen.py`：负责屏幕截图
### `system/context.py`

负责获取系统上下文，目前主要包含：

- 当前活动窗口标题
- 前台应用名粗略推断
- 空闲输入时长
- 最近 5 分钟窗口切换次数

平台实现策略：

- Ubuntu/Linux：`xdotool` + `xprintidle`
- Windows：Win32 API

### `model/client.py`

负责：

- 构造多模态模型请求
- 兼容本地 OpenAI 风格接口
- 对模型非标准 JSON 输出做提取
- 模型不可用时退回启发式判断

### `storage/db.py`

负责：

- 初始化 SQLite 表
- 写入每次观测结果
- 汇总最近样本
- 汇总日维度数据

### `reporting/`

- `scheduler.py`：判断现在是否应该生成日报
- `reporter.py`：把日报统计结果渲染成 Markdown

### `cleanup.py`

负责清理超出保留时间的旧截图。

### `agent.py`

负责把采集、推理、存储、日报调度串起来。

## 数据流

### 输入

- 屏幕截图
- 前台窗口标题
- 最近历史摘要
- 模型配置

### 输出

- SQLite 记录
- 控制台状态输出
- `reports/` 中的日报文件
- 被定期清理后的截图缓存目录

## 推荐扩展点

- 加入更强的窗口/浏览器上下文提取
- 加入结构化日志
- 加入 Web UI 或桌面 UI
- 加入更细的状态机
- 加入事件提醒与消息通知
